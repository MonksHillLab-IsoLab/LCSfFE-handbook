# Contributing

Thank you for your interest in improving this handbook.

The full contributor guide lives in [contributing.qmd](contributing.qmd). It explains:

- how to prepare content in Quarto Markdown
- how to contribute through pull requests
- how publishing to GitHub Pages works after changes are merged
- how maintainers create versioned releases that are archived in Zenodo

If you are reading the published handbook website or PDF, look for the `Contributing` chapter for the rendered version of the same guidance.

For substantial changes, please contribute through a pull request rather than editing published output files directly.
